### 1. Crearea Schemei (Executată ca `SYSTEM`)

Dacă trebuie să explici cum ai pregătit mediul, acesta este codul inițial. Schema este practic utilizatorul care deține datele.

```sql
-- Crearea utilizatorului/schemei dedicate
CREATE USER SMARTCITY IDENTIFIED BY parola_ta_aici;

-- Acordarea drepturilor necesare pentru a lucra
GRANT CONNECT, RESOURCE, UNLIMITED TABLESPACE TO SMARTCITY;

-- Ne mutăm pe noua schemă pentru a crea tabelele acolo
```

### 2. Structura Finală a Tabelelor (Executată ca `SMARTCITY`)

Acesta este scriptul „robust”. Am inclus toate optimizările pe care le-am făcut (mărirea coloanelor și schimbarea tipurilor de date pentru ID-uri) pentru a asigura un import fără erori.

```sql
-- ==========================================================
-- SURSA SQL 1: INFRASTRUCTURĂ TRANSPORT (Oracle)
-- ==========================================================

-- 1. Tabelul pentru stații
CREATE TABLE BUS_STOPS (
    STOP_ID NUMBER PRIMARY KEY,
    STOP_NAME VARCHAR2(500), -- Dimensiune mărită pentru nume lungi de stații
    STOP_LAT NUMBER,
    STOP_LON NUMBER
);

-- 2. Tabelul pentru rute
CREATE TABLE BUS_ROUTES (
    ROUTE_ID VARCHAR2(200) PRIMARY KEY, -- VARCHAR2 pentru a evita erorile de precizie (ORA-01438)
    ROUTE_SHORT_NAME VARCHAR2(200),
    ROUTE_LONG_NAME VARCHAR2(1000),     -- Spațiu generos pentru descrieri de traseu
    ROUTE_TYPE VARCHAR2(100)
);

-- 3. Tabelul de legătură (Missing Link între stații și rute)
CREATE TABLE ROUTE_STOPS (
    ROUTE_ID VARCHAR2(200),
    STOP_ID NUMBER,
    STOP_SEQUENCE NUMBER,
    -- Constrângerile se pot activa după import pentru integritate
    CONSTRAINT FK_ROUTE FOREIGN KEY (ROUTE_ID) REFERENCES BUS_ROUTES(ROUTE_ID),
    CONSTRAINT FK_STOP FOREIGN KEY (STOP_ID) REFERENCES BUS_STOPS(STOP_ID)
);

-- ==========================================================
-- SURSA DOCUMENT: STATISTICI ANALITICE (CSV Import)
-- ==========================================================

-- 4. Tabelul pentru statistici lunare (Sursa Eterogenă Document.DS)
CREATE TABLE TRANSPORT_STATS (
    STAT_ID NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    REPORT_MONTH VARCHAR2(100), -- Importat ca text pentru a evita conflictele de format DATE
    TRANSIT_RIDERSHIP NUMBER,   -- Volum pasageri
    GASOLINE_PRICE NUMBER,      -- Prețul combustibilului (corelație)
    ON_TIME_PERF NUMBER         -- Eficiența serviciului
);
```

### Justificări Tehnice pentru Documentație

Când prezinți proiectul, poți adăuga aceste note pentru a arăta că ai înțeles procesul de **Data Engineering**:

- **Flexibilitatea ID-urilor:** Am ales `VARCHAR2(200)` pentru `ROUTE_ID` deoarece datele externe (GTFS/Kaggle) pot avea formate imprevizibile, iar o bază de date de integrare (P1) trebuie să fie permisivă la ingestie.
- **Optimizarea pentru Analiză:** Tabelul `TRANSPORT_STATS` a fost simplificat masiv față de fișierul original (peste 130 coloane) pentru a păstra doar indicatorii relevanți (KPIs) pentru un model de tip Smart City.
